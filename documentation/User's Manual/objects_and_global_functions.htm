4.3.1. Name syntax

All object and function names in the EE scripting system follow certain rules to help the programmer to differentiate between his own objects and the engine's objects and also to tell what kind of object he is dealing with. These rules are:

  • Native objects with ETH* prefix are the ones that interact directly with the core engine, they may directly manipulate core engine objects.

  • All util, helper and data storage object names start with lower-case letters (e.g.: file, enmlFile, dateTime and collisionBox).

  • All native global functions start with upper-case letters, except for the basic math functions.

4.3.2. Objects

ETHEntity - entity object

Reference object that describes an entity in the scene. A reference object must always be a @handle, never an instance. It allows entity manipulation.

The most common way of getting a handle to an entity is to use entity exclusive callbacks, but they may also be retrieved with functions like SeekEntity.

While using an ETHEntity handle, make sure that the entity still exists, because if the program tries to manipulate an entity that has already been deleted the program will crash. Avoid storing ETHEntity handles externally.

Examples of ETHEntity usage:

// Moves the entity to the left
void ETHCallback_bullet(ETHEntity @thisEntity)
{
     thisEntity.AddToPosition(vector3(10,0,0));
}

 

// finds the starting point in the map
ETHEntity @circle = SeekEntity("start_point.ent");
if (circle !is null)
{
    // Adds the main character to the scene, places
    // him at the start point and gives him full HP
    int id = AddEntity("character.ent", circle.GetPosition(), 0.0f); // file name, position, angle
    ETHEntity @character = SeekEntity(id);
    character.SetFloat("hp", 100.0f);
}

ETHInput - input devices handler

Games may use the methods of the ETHInput object to perform keyboard, mouse or joystick input tests. This is a reference object and must always be a @handle, never an instance.

A handle to a valid ETHInput object may be retrieved at any time with the GetInputHandle function.

Examples of usage:
 
// if the user presses ESC, then quit
ETHInput @input = GetInputHandle();
if (input.KeyDown(K_ESC))
{
    Exit();
}

 

// gets the mouse cursor position
ETHInput @input = GetInputHandle();
vector2 cursorPos = input.GetCursorPos();

ETHEntityArray - dynamic entity array object

This object stores an array of entity handles (similar to pointers). The GetEntityArray function may be used to build custom arrays from entities in scene. More about entity arrays.

Example:

ETHEntityArray g_Soldiers;

void StartGameLogic()
{
    // gets the handles to each "soldier.ent" in scene
    GetEntityArray("soldier.ent", g_Soldiers);
}

void GameLoopLogic()
{
    // loops through every "soldier.ent" in scene
    const uint size = g_Soldiers.size();
    for (uint t=0; t<size; t++)
    {
        const vector2 v2Pos = g_Soldiers[t].GetPositionXY();
        g_Soldiers[t].AddToPositionXY(MoveSoldier());
    }
}

Methods:

uint ETHEntityArray::size() const
 -Returns the number of elements in the array.

void ETHEntityArray::clear()
 -Clears the array releasing all its handles.

void ETHEntityArray::unique()
 -Remove repeated entities from the array.

void ETHEntityArray::removeDeadEntities()
 -Remove all references to entities that have already been deleted from scene.

void ETHEntityArray::push_back(const ETHEntity &in)
 -Adds an entity handle to the end of the array.

void ETHEntityArray::operator+=
 -Add-assigns another array to the end of the current array. Example:

ETHEntityArray orcs;
GetEntityArray("orc.ent", orcs);

ETHEntityArray elfs;
GetEntityArray("elf.ent", elfs);

soldiers += orcs;
soldiers += elfs;
// now the "soldiers" array holds all elf and orc handles

// Clears the "orcs" and "elfs" arrays but the "soldiers" array still stores the handles
orcs.clear();
elfs.clear();

 

dictionary - hash map with string key

The following description and example was extracted from AngelScript's original documentation and source-code comments:

The dictionary object maps string values to values or objects of other types.

-Public script interface:

class dictionary
{
    void set(const string &in key, ? &in value);
    bool get(const string &in value, ? &out value) const;

    void set(const string &in key, int64 &in value);
    bool get(const string &in key, int64 &out value) const;

    void set(const string &in key, double &in value);
    bool get(const string &in key, double &out value) const;

    bool exists(const string &in key) const;
    void delete(const string &in key);
    void deleteAll();
}

-Script example:

dictionary dict;
obj object;
obj @handle;

dict.set("one", 1);
dict.set("object", object);
dict.set("handle", @handle);

if( dict.exists("one") )
{
    bool found = dict.get("handle", @handle);
    if( found )
    {
        dict.delete("object");
    }
}

dict.deleteAll();

 

dateTime - system date and time retriever

The dateTime object retrieves the system date and time automatically as soon as it is instantiated (its default constructor calls the ::update method):

-object public scope
class dateTime
{
    uint update()
    uint getDay() const
    uint getMonth() const
    uint getYear() const
    uint getHours() const
    uint getMinutes() const
    uint getSeconds() const
}

dateTime properties won't be kept automatically up-to-date. To reutilize an instance, call the update method first.

Sample:

dateTime dt;
print("The time is " + dt.getHours() + ":" + dt.getMinutes() + "\n");
print("The day is " + dt.getDay());

...
...

dt.update();
print("The time is " + dt.getHours() + ":" + dt.getMinutes() + "\n");
print("The day is " + dt.getDay());

 

enmlFile and enmlEntity - ENML (Easy Non-Markup Language) parsers

ENML is an human readable parsing language which is very similar to INI files but with a different syntax. It separates its values into global entity scopes, and their attributes are all based on strings.

These objects are very useful for more specific save game data, such as high scores, user data and preferences, and more. It was designed to be very easy to write and read and as it may work directly with strings, the programmer may implement his own cryptography algorithm to protect the data.

Objects:

class enmlEntity
{
    // clear all data in the entity
    void clear();

    // adds a new attribute to the entity
    void add(const string &nameKey, string attribute);

    // gets an attribute string
    string get(const string &nameKey);

    // returns a attribute name list separated by commas
    string getAttributeNames() const;
}

class enmlFile
{
    // clears all files in it
    void clear();

    // adds an entity to the file parser
    void addEntity(const string &entityName, const enmlEntity &in);

    // generates an ENML file string
    string generateString() const;

    // reads an ENML file string and parses its values
    uint parseString(const string &str);

    // returns true if the entity exists
    bool exists(const string &entityName) const;

    // Returns the value of the 'attribute' in the 'entity' in the file
    string get(const string &entity, const string &attribute) const;

    // parses numerical attributes in the file
    bool getDouble(const string &entity, const string &attrib, double &out) const;
    bool getInt(const string &entity, const string &attrib, int &out) const;
    bool getUInt(const string &entity, const string &attrib, uint &out) const;
    bool getFloat(const string &entity, const string &attrib, float &out) const;

    // returns the last parsing error message
    string getErrorString() const;

    // returns a string containing all entity names separated by commas
    string getEntityNames() const;

    // returns a string containing all attributes in the 'entity' separated by commas
    string getAttributeNames(const string &entity) const;

    // writes the internal data to a file
    void writeToFile(const string &fileName) const;

    // reads an ENML file and parses its data
    bool parseFromFile(const string &fileName);
}

Tutorial:

The syntax:

[test.txt]
// my first entity
theWall
{
    // each attribute's value ends with a ';' character.
    artist = Pink Floyd;
    // use '\;' to write ';' characters into the attribute
    formation = David Gilmour\; Nick Mason\; Roger Waters\; Bob Klose;
    year = 1979;
}

// my second entity
myCar
{
    name = Beetle;
    year = 1978; cc = 1500;
    Manufacturer = Volkswagen;

    // it also supports multi-line values:
    sucessorList =
        Volkswagen Golf
        Volkswagen Jetta (Sedan)
        Volkswagen New Beetle
        Volkswagen Gol (in Brazil); // don't forget to finish with ';'
}

numbers
{
    pi = 3.1415;
    posX = 1.5;
    myInteger = 13;
}

Reading an ENML file and retrieving data:

const string str = GetStringFromFile("test.txt");

enmlFile f;
f.parseString(str);
string name = f.get("myCar", "name");
string artist = f.get("theWall", "artist");

double myPi;
f.getDouble("numbers", "pi", myPi);

float posX;
f.getFloat("numbers", "posX", posX);

uint myInteger;
f.getUInt("numbers", "myInteger", myInteger);

Writing an ENML file:

enmlFile f;

enmlEntity entity;
entity.add("class", "rogue");
entity.add("level", "85");
entity.add("name", "Nathan");
f.addEntity("partyMember0", entity);

entity.clear();
entity.add("class", "paladin");
entity.add("level", "71");
entity.add("name", "John");
f.addEntity("partyMember1", entity);

string str = f.generateString();
SaveStringToFile("myFile.txt", str);

That will produce:

partyMember0
{
    class = rogue;
    level = 85;
    name = Nathan;
}

partyMember1
{
    class = paladin;
    level = 71;
    name = John;
}

Things it cannot do:
-Can't have entity trees (entities inside another entity). All entities MUST be global.
-Values that start with a return character.
-Receive attributes with empty values

Remarks:
-In the files, use '\\' and '\;' to write '\' and ';' characters to attribute files
-Use '/' for comments. It won't work inside value definitions.

ETHEntity @DeleteEntity(ETHEntity &in)
 -Deletes the entity passed as parameter and returns null if it has been successfully deleted, otherwise it will return a handle to the object itself. For safety reasons, it is recommended to assign the return value directly to the handle, so will will be null if the entity no longer exists. This call removes the entity from the scene, be it wont guarantee that the memory will be freed as long as there are other handles alive. For this reason, avoid having more than one handle to the same entity (specially if they are global). Example:

@character = DeleteEntity(character);
if (character is null)
{
    print("It has been successfully deleted\n");
}

bool GetEntityArray(const string &name, ETHEntityArray &outArray)
 -Creates an array of entity handles (pointers) containing every entity whose name match name and stores the array in the ETHEntityArray object. This function is accumulative, it means that previous data in the array will be kept, even repeated items. Example of usage:

// Get all fire.ent handles
GetEntityArray("fire.ent", fireEntities);
const uint numFire = fireEntities.size();

// Loops through all "fire.ent" entities
for (uint t=0; t<numFire; t++)
{
    // convert them to dynamic entities
    fireEntities[t].TurnDynamic();
}
   
// Now release the handles
fireEntities.clear();

 -Draws a text string to the screen. This function supports bitmap fonts exported by the AngelCode's BMFont tool (*.fnt files on text mode). The fonts may use multiple bitmap pages, but it is highly recommended to use only one page for better performance.
The *.fnt font files and all its pages must be in the "{project_path}/data/" folder or its equivalent package. Ethanon Engine includes to the projects some default bitmap fonts (that were generated with the AngelCode's BMFont):

-{project_path}/data/Verdana14_shadow.fnt
-{project_path}/data/Verdana20.fnt

-{project_path}/data/Verdana30.fnt
-{project_path}/data/Verdana30_shadow.fnt
-{project_path}/data/Verdana64.fnt
All these font files are included by default but the user may create and edit his own fonts with BMFont.

// text drawing example
DrawText(vector2(0,200), "Text drawing test!", "Verdana14_shadow.fnt", ARGB(250,255,255,255));
